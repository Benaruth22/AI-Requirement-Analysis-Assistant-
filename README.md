# AI Requirement Analysis Assistant

An AI-powered application that automates Software Requirement Specification (SRS) analysis using Large Language Models (LLMs) and Natural Language Processing (NLP). The system extracts software requirements, detects ambiguities, and generates structured user stories and acceptance criteria, helping teams accelerate requirement engineering and improve software quality.

---

## Key Features

Analyze Software Requirement Specification (SRS) documents

Extract functional and non-functional requirements

Detect ambiguous, incomplete, and duplicate requirements

Generate user stories automatically

Generate acceptance criteria for Agile development

Produce structured AI-generated outputs

Reduce manual requirement analysis effort

Improve software requirement review efficiency

## How It Works

1. Upload a Software Requirement Specification (SRS) document.
2. Extract and preprocess document content.
3. Analyze the requirements using an LLM.
4. Identify ambiguities, inconsistencies, and missing information.
5. Generate Functional Requirements, Non-Functional Requirements, User Stories, and Acceptance Criteria.
6. Display structured results for review.
